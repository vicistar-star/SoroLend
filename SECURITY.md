# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (`main`) | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report security issues via one of these channels:

- **Email:** security@sorolend.finance  
- **Bug Bounty:** https://immunefi.com/bounty/sorolend (preferred for smart contract bugs)

Include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within **48 hours** and aim to provide a fix or mitigation within **7 days** for critical issues.

## Bug Bounty

| Severity | Reward |
|----------|--------|
| Critical | Up to $100,000 |
| High | Up to $30,000 |
| Medium | Up to $5,000 |
| Low | Up to $1,000 |

Full scope and rules are listed on [Immunefi](https://immunefi.com/bounty/sorolend).

## Disclosure Policy

We follow coordinated disclosure. Please give us reasonable time to patch before public disclosure. We will credit reporters in our release notes unless anonymity is requested.
